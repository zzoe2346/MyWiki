- 내부의 기능적 제약사항
	- 대부분의 조인이 중첩 루프 조인 알고리즘으로 수행
	- 상용 DBMS와 다르게 수행된 쿼리 결과가 메모리에 적재되는 캐시 기능에 한계
		- 데이터 변경시 캐시 모두 삭제되는 등으로 일반적 쿼리 작성 및 튜닝 안될수도
	- 따라서, 
		- DBMS기능 자세히 알아야되고
		- 실행 계획 정보 해석하고 문제점 인지하고 대응 능력 필요
- 강점
	- 무료
	- 경량화
- 단점
	- 수행 가능 알고리즘 적어 성능적 불리
- 대신 다양한 스토리지 엔진 적극적으로 사용할 기회 존재
